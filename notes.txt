Contraints 
--------------
>> Constraints are rules applied to a table column to maintain data integrity and accuracy.

1) PRIMARY KEY --> UNIQUE, NOT NULL. It is used to identify each rows uniquely. Only one primary key per table.

2) AUTO_INCREMENT --> Automatically increases the numeric value by 1 for every new record.

3) NOT NULL --> ensures a column cannot have NULL value.

4) UNIQUE --> ensures all values in a column are different.

5) DEFAULT --> assigning default value for a column.

6) CHECK --> Ensures a column's value meet a condition

7) FOREIGN KEY --> A foreign key in a table  refers to primary key in another table.
Ex: foreign key (col) reference (col)

Joins
-------
>> A join in MySQL is used to combine data from two or more tables based on related column.

Inner Join
-----------
>> It returns only matching rows from both tables.
Ex: select col det FROM tb1 INNER JOIN tb2 ON exp;

Left Join
-----------
>> It returns all rows from left. Matching rows from right table. Null where no match found.
Ex: select col det FROM tb1 LEFT JOIN tb2 ON exp;

Right Join
-----------
>> It returns all rows from right. Matching rows from left table. Null where no match found.
Ex: select col det FROM tb1 RIGHT JOIN tb2 ON exp;

CROSS JOIN 
------------
>> No need for related columns. matchs all rows from left table to all from right table.
Ex: SELECT * FROM tb1 CROSS JOIN tb2;


Aggregate Functions
--------------------
>> Aggregate Functions performs a calculation on a set of rows and return a single value.

COUNT --> no of records. no of values in a column(ignore null);
AVG(col) --> average value of a numeric column.
MAX(col) --> returns maximum value.
MIN(col) --> returns minimum value.
SUM(col) --> returns the sum of numeric column.

GROUP BY 
----------
>> It groups rows having the same vale.
Ex: select category, count(*) as noOfProducts, sum(price) as totalPrice,
avg(price) as averagePrice from products group by category;
